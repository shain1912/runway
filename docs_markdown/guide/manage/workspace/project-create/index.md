# 프로젝트 생성

프로젝트는 Runway 플랫폼에서 리소스를 팀/기능 단위로 구분하고 관리할 수 있도록 돕는 핵심 단위입니다. 각 프로젝트는 컴퓨팅 자원(CPU, 메모리, 디스크, GPU 등)의 사용 한도를 설정할 수 있으며, 워크스페이스 내에서 독립된 작업 환경을 구성하는 데 사용됩니다.

> Runway 워크스페이스 > **프로젝트** 메뉴

> **Note**: 프로젝트 생성/수정/삭제 권한
>
> - **생성**: **워크스페이스 관리자(Workspace Admin)**와 **워크스페이스 멤버(Workspace Member)**가 생성할 수 있습니다.
> - **수정/삭제**: **워크스페이스 관리자(Workspace Admin)**이거나 해당 프로젝트의 **프로젝트 관리자(Project Admin)** 권한이 있어야 가능합니다.
>
> 사용자 역할은 워크스페이스 > **멤버** 메뉴의 **역할** 열에서 확인할 수 있습니다.
>
> 

---

## 프로젝트 생성하기

**워크스페이스 관리자**와 **워크스페이스 멤버**는 워크스페이스 내에서 새로운 프로젝트를 생성할 수 있습니다.  
프로젝트를 생성할 때는 팀의 작업 목적과 필요한 리소스 범위를 고려하여 **이름과 설명을 명확하게 작성**하는 것이 좋습니다.

1.  워크스페이스에서 **프로젝트** 메뉴로 이동합니다. 

    

2.  **전체 프로젝트** 영역 오른쪽 위치한 **+ 생성** 버튼을 클릭합니다.

3.  오른쪽 슬라이드 패널에서 **일반 세부 정보**를 입력합니다.

    
    
    - **이름**: 프로젝트 이름을 입력합니다.(최대 128자)

    - **ID**: 입력한 프로젝트 이름에 따라 자동 추천되며, 생성 시에는 수정할 수 있습니다. (3-31자, 영문 소문자, 숫자, 하이픈(-)만 사용 가능)

        > **Info**: 프로젝트 ID 생성 규칙
        > 프로젝트 ID는 프로젝트의 고유 식별자로, **생성 후에는 변경할 수 없으므로** 신중하게 설정해야 합니다.
        >
        > ** &nbsp;ID 구성 방식**
        >
        > - 워크스페이스 ID 및 구분자(최대 9자)가 접두사로 자동 추가됩니다.(변경 불가)
        > - 워크스페이스 ID 뒤에 프로젝트 이름 기반으로 자동 추천되나 프로젝트 생성 시에는 수정 가능합니다.
        >
        > {width=60%}
        >
        > ** &nbsp;ID 규칙**
        >
        > 프로젝트 ID는 Kubernetes 네임스페이스명으로 사용되므로 다음 규칙을 준수해야 합니다.
        >
        > - **길이**: 최소 3자, 최대 31자(입력 가능 값 기준)
        > - **허용 문자**: 소문자 영문(a-z), 숫자(0-9), 하이픈(-)
        > - **시작/끝**: 소문자 또는 숫자 (하이픈 불가)
        >
        > ** &nbsp;권장사항**
        >    
        > 짧고 의미가 명확한 ID로 설정하면 관리가 편리합니다.

    - **설명**(선택): 프로젝트에 대한 간단한 설명을 입력합니다.

4.  **리소스 할당량**을 설정합니다.

    이 단계에서는 **프로젝트 단위에서 사용할 수 있는 최대 컴퓨팅 자원 한도**를 설정합니다.  
    설정한 값은 이 프로젝트에 생성되는 모든 작업(학습, 추론, 애플리케이션 실행 등)의 상한선으로 적용됩니다.

    - 각 리소스 항목(CPU, 메모리, 디스크, GPU)에 대해 프로젝트가 사용할 수 있는 최대 할당량을 지정합니다.
    - 프로젝트의 리소스 할당량은 **소속 워크스페이스에 설정된 자원 한도 범위 내에서만** 설정할 수 있습니다.

    > **Info**: 최대 자원량 확인
    >
    > 각 입력 필드 오른쪽에는 현재 이 프로젝트에서 **설정 가능한 최대 자원량**이 표시됩니다.  
    > 이 값은 워크스페이스에 할당된 자원 한도와 현재 사용 상황을 기준으로 자동 계산됩니다.
    >
    > - **CPU**: 프로젝트에서 사용할 수 있는 최대 코어 수
    > - **메모리**: 프로젝트에서 사용할 수 있는 최대 메모리 용량
    > - **디스크**: 프로젝트에서 사용할 수 있는 최대 디스크 용량
    > - **GPU**: 프로젝트에서 사용할 수 있는 GPU 수  
    >   (워크스페이스에 할당된 노드에 따라 자동으로 표시됩니다. GPU가 할당되지 않은 경우 0으로 표시됩니다.)

5.  **생성** 버튼을 클릭하여 프로젝트를 생성을 완료합니다.
