# 프로젝트 사용자 관리

프로젝트 사용자는 역할에 따라 접근 가능한 기능이 다릅니다.  
프로젝트 관리자는 사용자 추가, 참여 요청 승인, 역할 변경, 제거를 수행할 수 있습니다.

> Runway 워크스페이스 > 프로젝트 > **사용자** 메뉴

이 페이지에서 할 수 있는 작업은 다음과 같습니다.

-    **프로젝트 내 역할 및 권한 이해**

    ---

    멤버를 추가하기 전에 프로젝트 역할 체계와 권한 범위를 확인합니다.

     [역할 및 권한 이해](#project-user-structure)

-    **멤버 조회**

    ---

    프로젝트 멤버 목록과 역할, 상세 정보를 확인합니다.

     [멤버 조회하기](#view-project-users)

-    **멤버 추가**

    ---

    워크스페이스 사용자를 프로젝트 멤버로 추가하고 역할을 지정합니다.

     [멤버 추가하기](#add-project-users)

-    **참여 요청 승인**

    ---

    워크스페이스 사용자의 프로젝트 참여 요청을 승인하거나 거절합니다.

     [참여 요청 승인하기](#approve-project-requests)

-    **역할 변경**

    ---

    프로젝트 멤버의 역할을 변경하거나 관리자 권한을 이전합니다.

     [역할 변경하기](#change-project-user-role)

-    **멤버 제거**

    ---

    프로젝트에서 멤버를 제거합니다.

     [멤버 제거하기](#remove-project-user)

## 사용자 역할 및 권한 이해

프로젝트에 사용자를 추가하거나 역할을 변경하기 전에 프로젝트 역할 체계를 이해해야 합니다.  
프로젝트 내 리소스 관리, 애플리케이션 실행 등의 실무 권한은 **프로젝트 역할**에 의해 결정되며, 워크스페이스 역할과는 독립적으로 프로젝트별로 지정됩니다.

> **Note**: 상속된 관리자 권한(Mixed Roles) 표시
> 
> 멤버 목록의 역할 열에 **Mixed roles** 마크가 표시되는 경우, 해당 사용자가 **플랫폼 관리자** 또는 **워크스페이스의 관리자** 권한을 보유하고 있어 
> 표시된 워크스페이스/프로젝트 역할보다 상위 권한을 상속받고 있음을 의미합니다. **Mixed roles**가 표시된 경우 프로젝트의 역할과 관계없이 **프로젝트에 대한 모든 권한**을 가집니다.
>
>

### 프로젝트 역할의 종류

프로젝트에 사용자를 추가할 때 아래 세 가지 역할 중 하나를 지정합니다.

| 역할 | 설명 |
|----|----|
| **관리자**(Admin) | 프로젝트 내 모든 관리 권한 보유 |
| **멤버**(Member) | 사용자 및 역할 관리 기능을 제외한 대부분의 주요 기능 사용 가능 |
| **뷰어**(Viewer) | 모델 및 서비스, 컴퓨팅 리소스 현황 등 조회만 가능 |

> **Info**: 사용자 정의 역할
> 
> 기본 역할(관리자, 멤버, 뷰어) 외에 **권한을 조합한 사용자 정의 역할**을 만들어 부여할 수 있습니다.
> 사용자 정의 역할과 권한 조합에 대한 방법은 **[프로젝트 사용자 역할 관리](project-roles.md)**를 참고하세요.

---

### 프로젝트 역할별 권한

프로젝트 역할별 권한은 다음과 같습니다.

| 역할 | 프로젝트 관리 | 사용자 관리 | 리소스 관리 | 애플리케이션 관리 |
|---|---|---|---|---|
| **관리자**(Admin) | 수정/삭제 | 추가/수정/제거 | 수정 | 생성/수정/삭제 |
| **멤버**(Member) | 조회 | **추가** | 조회 | **생성/수정/삭제** |
| **뷰어**(Viewer) | 조회 | 조회 | 조회 | 조회 |

> **Note**: 프로젝트 역할과 워크스페이스 역할의 관계
> 
> 프로젝트에 추가할 수 있는 사용자는 **해당 워크스페이스에 소속된 사용자**에 한합니다. 워크스페이스 역할(관리자/멤버/뷰어)은 워크스페이스 수준의 권한을 결정하며, 프로젝트 내 실무 권한과는 별개입니다.  워크스페이스 역할에 대한 자세한 내용은 **[워크스페이스 사용자 관리](../../manage/workspace/workspace-users.md)**를 참고하세요.

## 프로젝트 사용자 조회하기

**모든 프로젝트 멤버**는 프로젝트 내 사용자의 목록과 상세 정보를 확인할 수 있습니다.

1.  프로젝트 페이지에서 **사용자** 메뉴로 이동합니다.

    

2.  **사용자** 탭에서 검색하거나 역할 필터 등을 사용하여 조회할 사용자를 찾습니다.

3.  사용자 이름을 클릭하여 상세 정보를 확인합니다.
    

    - **프로필**: 이메일, 이름, 성, 전화번호, 가입일
    - **역할**: 현재 프로젝트 역할 (프로젝트 관리자인 경우 **편집** 버튼으로 역할 변경 가능)

## 프로젝트 사용자 추가하기

프로젝트 관리자는 워크스페이스 내 사용자를 프로젝트 사용자로 추가할 수 있습니다.

1.  프로젝트 페이지에서 **사용자** 메뉴로 이동합니다.

    

2.  **사용자** 탭에서 오른쪽의 **+ 추가** 버튼을 클릭합니다.

3.  추가할 사용자에 체크합니다.

    

4.  필요에 따라 선택한 사용자의 **역할**을 변경합니다.

    | 역할 | 설명 |
    |----|----|
    | **관리자**(Admin) | 프로젝트 내 모든 관리 권한 보유 |
    | **멤버**(Member) | 사용자 및 역할 관리 기능을 제외한 대부분의 주요 기능 사용 가능 |
    | **뷰어**(Viewer) | 모델 및 서비스, 컴퓨팅 리소스 현황 등 조회만 가능 |

5.  **추가** 버튼을 클릭하여 사용자를 추가합니다.

---

## 프로젝트 참여 요청 승인하기

워크스페이스 사용자가 프로젝트 참여를 요청하면 프로젝트 관리자(Admin)가 요청을 승인하여 프로젝트 사용자를 추가할 수 있습니다.

> **Note**: 프로젝트 참여 요청 승인 권한
>
> 프로젝트 역할이 **관리자(Admin)**인 사용자만 승인/거절을 수행할 수 있습니다.

1.  프로젝트 페이지에서 **사용자** 메뉴로 이동합니다.

    

2.  **가입 요청** 탭에서 요청 목록을 확인합니다.

3.  요청 행의 ⋮ 더 보기 버튼을 클릭하고 **승인**을 선택합니다.

4.  가입을 승인할 사용자의 **역할**을 선택합니다.

    

5.  **승인** 버튼을 클릭합니다.

> **Info**: 참여 요청 거절
>
> ⋮ 더 보기 버튼에서 **거절**을 선택할 수도 있습니다. 거절을 선택하면 거절 사유를 입력한 뒤 **거절** 버튼을 클릭합니다.
>
>

## 프로젝트 사용자 역할 변경하기

프로젝트 관리자는 프로젝트 내에서 사용자의 역할을 변경할 수 있습니다.

1.  프로젝트 페이지에서 **사용자** 메뉴로 이동합니다.

    

2.  **사용자** 탭에서 역할을 변경할 사용자 이름을 클릭합니다.

3.  **역할** 영역의 **편집** 버튼을 클릭합니다.

    

4.  변경할 역할을 선택하고 **저장** 버튼을 클릭합니다.

    

    | 역할 | 설명 |
    |----|----|
    | **관리자**(Admin) | 프로젝트 내 모든 관리 권한 보유 |
    | **멤버**(Member) | 사용자 및 역할 관리 기능을 제외한 대부분의 주요 기능 사용 가능 |
    | **뷰어**(Viewer) | 모델 및 서비스, 컴퓨팅 리소스 현황 등 조회만 가능 |

## 프로젝트 사용자 제거하기

프로젝트 관리자는 사용자를 프로젝트에서 제거할 수 있습니다.

1.  프로젝트 페이지에서 **사용자** 메뉴로 이동합니다.
    
    

2.  **사용자** 탭에서 삭제할 사용자 행의  (더 보기) > **제거**를 선택합니다.

3.  확인 팝업에서 **제거** 버튼을 클릭합니다.
