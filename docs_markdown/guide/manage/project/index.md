# 프로젝트 관리

프로젝트는 워크스페이스 내에서 MLOps 워크플로우를 실행하는 단위입니다.
프로젝트 사용자는 역할에 따라 애플리케이션을 실행하고, 사용자를 관리하며, 리소스를 운영할 수 있습니다.

> Runway 워크스페이스 > 프로젝트 > 좌측 네비게이션 메뉴

---

## 주요 내용

-    **사용자 관리**

    ---

    프로젝트에 사용자를 추가하고, 역할을 변경하거나 제거합니다.

     [프로젝트 사용자 관리](project-users.md)

-    **역할 관리**

    ---

    시스템 기본 역할을 확인하고, 권한을 조합한 사용자 정의 역할을 생성·수정·삭제합니다.

     [프로젝트 역할 관리](project-roles.md)

-    **리소스 모니터링**

    ---

    프로젝트 단위의 리소스 사용 현황과 워크로드 상태를 확인합니다.

     [프로젝트 리소스 모니터링](../../operate/monitoring/project-monitoring.md)

-    **리소스 변경**

    ---

    프로젝트의 컴퓨팅 리소스 할당량을 조정합니다.

     [프로젝트 리소스 변경](project-resource.md)

-    **이름 변경**

    ---

    프로젝트의 이름과 설명을 수정합니다.

     [프로젝트 이름 변경](project-info.md)

-    **프로젝트 삭제**

    ---

    프로젝트를 삭제합니다.

     [프로젝트 삭제](project-delete.md)

---

## 역할별 주요 작업

| 작업 | 메뉴 위치 | 수행 가능한 역할 | 설명 |
|---|---|---|---|
| 사용자 추가 | 사용자 | 관리자 | 워크스페이스 사용자를 프로젝트에 추가합니다. |
| 참여 요청 승인 | 사용자 | 관리자 | 프로젝트 참여 요청을 승인하거나 거절합니다. |
| 역할 변경 | 사용자 | 관리자 | 프로젝트 멤버의 역할을 변경합니다. |
| 역할 생성·관리 | 설정 > 역할 | 관리자 | 사용자 정의 역할을 생성·수정·삭제합니다. |
| 애플리케이션 관리 | 애플리케이션 | 관리자, **멤버** | **애플리케이션을 배포·실행·수정·삭제합니다.** |
| 리소스 모니터링 | 모니터링 | 관리자 | 프로젝트 리소스 사용 현황을 조회합니다. |
| 리소스 변경 | 설정 > 일반 | 관리자 | 프로젝트 리소스 할당량을 변경합니다. |
| 이름·설명 변경 | 설정 > 일반 | 관리자 | 프로젝트 이름과 설명을 수정합니다. |
| 프로젝트 삭제 | 설정 > 일반 | 관리자 | 프로젝트를 삭제합니다. |

> **Info**: 사용자 정의 역할
> 위 표는 시스템 기본 역할(관리자, 멤버, 뷰어) 기준입니다.
> **사용자 정의 역할**을 통해 관리자가 아닌 사용자에게도 개별 권한을 부여할 수 있습니다.
> 자세한 내용은  **[프로젝트 역할 관리](project-roles.md)**를 참고하세요.
