# 프로젝트 사용자 역할 관리

프로젝트의 역할은 사용자 권한을 제어하기 위해 제공하거나 관리자가 권한을 조합하여 생성하는 사용자 그룹(권한 세트)입니다.
역할은 **시스템에서 제공하는 기본 역할(System)**과 프로젝트 **관리자가 정의한 사용자 정의 역할(Custom)**로 나뉩니다.

> Runway 워크스페이스 > 프로젝트 > **설정** > **역할** 메뉴

> **Note**: 프로젝트 사용자 역할 관리 권한
>
> 프로젝트 역할이 **관리자(Admin)**인 사용자와 역할 수정 권한을 부여받은 사용자만 변경할 수 있습니다.
> 그 외 사용자는 조회만 가능합니다.

이 페이지에서 할 수 있는 작업은 다음과 같습니다.

-    **역할 종류 및 권한 이해**

    ---

    시스템 역할의 종류와 부여 가능한 권한 목록을 확인합니다.

     [역할 및 권한 이해](#project-role-types)

-    **역할 생성**

    ---

    사용자 정의 역할을 생성합니다.

     [역할 생성하기](#create-project-role)

-    **역할 수정**

    ---

    역할의 이름, 설명, 권한을 수정합니다.

     [역할 수정하기](#edit-project-role)

-    **역할 삭제**

    ---

    사용자 정의 역할을 삭제하고 대체 역할을 확인합니다.

     [역할 삭제하기](#delete-project-role)

## 역할의 종류와 설정 가능한 권한

### 시스템 제공 기본 역할

프로젝트에는 다음 세 가지 기본 역할이 제공됩니다. 기본 역할은 수정하거나 삭제할 수 없습니다.

| 역할 | 설명 |
|---|---|
| **관리자**(Admin) | 프로젝트의 모든 관리 권한을 가집니다.  보유 권한 : **모든 권한** |
| **멤버**(Member) | 애플리케이션 관리 등 프로젝트 내 주요 운영이 가능합니다.  보유 권한: **사용자 추가, 애플리케이션 템플릿, 애플리케이션, 추론 서비스 관련 모든 권한** |
| **뷰어**(Viewer) | 프로젝트 정보를 조회만 할 수 있습니다.  보유 권한 : **없음(조회만 가능)** |

---

### 조합 가능한 권한의 목록

사용자 정의 역할을 생성할 때 아래 권한을 조합하여 부여할 수 있습니다. 권한은 9개 카테고리로 구분됩니다.

> **Info**: 조회 권한 자동 부여
> `create` · `edit` · `delete` 권한 중 하나를 선택하면, 해당 카테고리의 `list`(목록 조회)와 `view`(상세 정보 조회) 권한이 자동으로 함께 부여됩니다.

| 카테고리 | 권한 | 설명 |
|---|---|---|
| **프로젝트** | `project.view` | 프로젝트 상세 정보 및 지표 조회 |
| | `project.edit` | 프로젝트 설정 및 구성 수정 |
| | `project.delete` | 프로젝트 삭제 |
| **멤버십** | `membership.create` | 프로젝트에 멤버 초대 |
| | `membership.edit` | 멤버 역할 및 권한 수정 |
| | `membership.delete` | 프로젝트에서 멤버 제거 |
| **역할** | `role.list` | 역할 목록 조회 |
| | `role.view` | 역할 상세 정보 조회 |
| | `role.create` | 사용자 정의 역할 생성 |
| | `role.edit` | 역할 권한 및 설정 수정 |
| | `role.delete` | 사용자 정의 역할 제거 |
| **애플리케이션 템플릿** | `application_template.list` | 애플리케이션 템플릿 목록 조회 |
| | `application_template.view` | 애플리케이션 템플릿 상세 정보 조회 |
| **애플리케이션** | `application.list` | 애플리케이션 목록 조회 |
| | `application.view` | 애플리케이션 상세 정보 조회 |
| | `application.create` | 애플리케이션 배포 및 실행 |
| | `application.edit` | 애플리케이션 설정 및 구성 수정 |
| | `application.delete` | 애플리케이션 제거 |
| **스토리지** | `storage.list` | 스토리지 목록 조회 |
| | `storage.view` | 스토리지 상세 정보 조회 |
| | `storage.create` | 새 스토리지 볼륨 생성 |
| | `storage.edit` | 스토리지 설정 수정 |
| | `storage.delete` | 스토리지 볼륨 제거 |
| **추론 엔드포인트** | `inference_endpoint.list` | 추론 엔드포인트 목록 조회 |
| | `inference_endpoint.view` | 추론 엔드포인트 상세 정보 조회 |
| | `inference_endpoint.create` | 추론 엔드포인트 생성 및 모델 배포 |
| | `inference_endpoint.edit` | 추론 엔드포인트 및 배포 모델 수정 |

| | `model_deployment.view` | 모델 배포 상세 정보 조회 |
| | `model_deployment.create` | 모델 배포 생성 |
| | `model_deployment.edit` | 모델 배포/해제 및 수정 |

## 사용자 정의 역할 생성하기

**프로젝트 관리자**(또는 **권한이 있는 사용자**)는 필요에 따라 권한을 조합한 **사용자 정의 역할**을 생성할 수 있습니다.

1.  프로젝트에서 **설정** > **역할** 메뉴로 이동합니다.

    

2.  오른쪽 상단의 **+ 생성** 버튼을 클릭합니다.

3.  역할의 **이름**과 **설명**을 입력합니다.

    

    - **이름**: 역할의 용도를 알 수 있도록 간결하게 작성합니다. (예: `리소스 관리자`, `앱 관리자`)
    - **설명**: 역할의 목적이나 대상 사용자를 명시합니다.

4.  **생성 방법**을 선택합니다.

    > **Info**: 생성 방법 선택
    >
    > 권한 조합의 편의를 위해 두 가지 생성 방법 중 하나를 선택할 수 있습니다.
    >
    > - **처음부터 생성**: 권한이 모두 해제된 상태에서 필요한 권한에만 체크하여 생성합니다.
    > - **기존 역할 조합**: 이미 존재하는 역할 중 하나 이상을 선택하여 **선택한 역할이 가진 권한에 체크된 상태**에서 필요한 권한을 추가하거나 해제하여 생성합니다.
    >
    > 

5.  권한 이름과 함께 표시되는 설명을 참고하여 부여할 **권한** 항목에 체크합니다.

    > **Info**: 조회 권한 자동 선택
    > `create` · `edit` · `delete` 권한을 선택하면 해당 카테고리의 `list`(목록 조회)와 `view`(상세 조회) 권한이 자동으로 함께 선택됩니다.

6.  화면 하단 **플랫폼 권한** 영역에서 각 플랫폼 앱에 부여할 역할을 선택합니다.

    

    - 설정 가능한 플랫폼 앱: **Gitea**, **OpenBao**, **Airflow**, **SeaweedFS**, **MLflow**, **Langfuse**
    - 각 앱별로 드롭다운에서 **Admin**, **Member**, **Viewer** 중 하나를 선택합니다.

    > **Warning**: 플랫폼 앱 역할 미설정 시
    >
    > 역할을 선택하지 않으면 해당 앱에 접근할 수 없습니다.  
    > 권한 선택이 표시되지 않는 플랫폼 앱의 경우, 조합된 권한에 따라 자동으로 매핑됩니다.

7.  **생성** 버튼을 클릭합니다.

> **Info**: 생성된 사용자 정의 역할의 활용
>
> 생성된 역할은 프로젝트에 사용자를 추가하거나 기존 사용자의 역할을 변경할 때 지정할 수 있습니다.
> 자세한 내용은 **[프로젝트 사용자 역할 변경하기](project-users.md#change-project-user-role)**를 참고하세요.
>
> {width="80%"}

## 사용자 정의 역할의 권한 수정하기

**프로젝트 관리자**(또는 **권한이 있는 사용자**)는 생성된 사용자 정의 역할의 이름, 설명과 부여된 권한을 수정할 수 있습니다.

> **Note**: 기본 역할 수정 제한
>
> 시스템 기본 역할(관리자/멤버/뷰어)은 권한을 수정할 수 없습니다.

1.  프로젝트에서 **설정** > **역할** 메뉴로 이동합니다.

    

2.  역할 목록에서 수정할 역할의 이름을 클릭합니다.

3.  역할 상세 페이지의 오른쪽 상단에 위치한  (더 보기) > **편집**을 선택합니다.

    

4.  필요에 따라 **권한**을 수정합니다.

    

    - **이름**과 **설명**도 함께 수정할 수 있습니다.
    - 권한 이름과 함께 표시되는 설명을 참고하여 부여할 **권한** 항목의 체크를 변경합니다.

5.  **저장** 버튼을 클릭합니다.

## 역할 삭제하기

**프로젝트 관리자**(또는 **권한이 있는 사용자**)는 생성된 사용자 정의 역할을 삭제할 수 있습니다.

1.  프로젝트에서 **설정** > **역할** 메뉴로 이동합니다.

    

2.  역할 목록에서 삭제할 역할의 이름을 클릭합니다.

3.  역할 상세 페이지의 오른쪽 상단에 위치한  (더 보기) > **삭제**를 선택합니다.

    

4.  삭제할 역할에 해당하는 사용자가 어떤 역할로 대체되는지 확인한 뒤 **삭제** 버튼을 클릭하면 역할이 삭제됩니다.
