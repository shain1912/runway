# 액세스 키 관리

액세스 키는 외부 애플리케이션, CLI, SDK 등에서 Runway 플랫폼에 접근할 때 인증을 위한 자격 증명으로 사용됩니다.
Runway에서는 용도에 따라 **API 키**, **LLM API 키**, **S3 키**의 세 가지 유형의 액세스 키를 제공합니다.

> 오른쪽 상단 프로필 아이콘 > **계정 설정** > **액세스 키** 메뉴

---

## 키 유형

액세스 키 메뉴에서는 다음 세 가지 탭을 통해 키를 관리할 수 있습니다.

| 키 유형 | 용도 | 주요 사용 사례 | 최대 개수 |
|---|---|---|---|
| **API 키** | Runway REST API 인증 | CLI/SDK를 통한 플랫폼 리소스 관리, CI/CD 파이프라인 연동, 외부 시스템에서의 어플리케이션 배포 자동화, Inference Endpoint·MLflow·Airflow 기능 연동 | 1개 |
| **LLM API 키** | LLM 추론 서비스 인증 | Runway에서 배포한 LLM 추론 엔드포인트 호출, OpenAI 호환 API 형식의 외부 클라이언트 연동 | 1개 |
| **S3 키** | SeaweedFS S3 스토리지 인증 | SeaweedFS S3 저장소의 모델 아티팩트·데이터셋 업로드/다운로드, 스토리지에 대한 프로그래밍 방식의 접근 (Access Key / Secret Key 쌍으로 제공) | 1개 |

### S3 키 상세 정보

Runway는 오브젝트 스토리지로 **SeaweedFS**를 사용합니다. S3 키 탭에서는 키 외에도 SeaweedFS S3 엔드포인트 연결에 필요한 **엔드포인트**, **리전** 정보와 버킷별 **권한** 정보를 함께 확인할 수 있습니다.

| 항목 | 설명 |
|---|---|
| **Endpoint** | SeaweedFS S3 스토리지의 접속 URL |
| **Region** | 스토리지 리전 정보 |

S3 키 탭 하단의 **권한** 영역에서는 현재 사용자가 접근 가능한 버킷 목록과 각 버킷에 대한 권한(Read, Write, List, Tagging, Admin)을 확인할 수 있습니다.

> **Info**: S3 클라이언트 연동
>
> SeaweedFS는 S3 호환 API를 제공하므로 AWS CLI, boto3 등 S3 호환 클라이언트를 그대로 사용할 수 있습니다. S3 키 탭에 표시된 **Endpoint**, **Region**, **Access Key**, **Secret Key** 정보를 클라이언트 설정에 입력하세요.

## 키 생성하기

키 생성 방법은 키 유형에 관계없이 모두 동일합니다.

1.  계정 설정 페이지에서 **액세스 키** 메뉴를 선택합니다.

    

2.  생성할 키 유형의 탭(**API 키**, **LLM API 키**, **S3 키**)을 선택합니다.

3.  **+ 생성** 버튼을 클릭합니다.

4.  생성된 키를 복사하여 안전한 곳에 보관합니다.

    

> **Warning**: 키 보관 및 보안 주의
>
> 생성된 키는 **생성 시점에만** 전체 값을 확인하고 복사할 수 있습니다. 이후 키 목록에서는 키의 일부만 마스킹되어 표시되며, 해당 키가 유효한지 식별하는 용도로만 참고할 수 있습니다. **키를 분실한 경우 기존 키를 삭제하고 새로 생성**해야 합니다. 키가 외부에 유출되지 않도록 안전한 곳에 보관하세요.

---

## 키 삭제하기

키 삭제 방법은 키 유형에 관계없이 모두 동일합니다.

1.  계정 설정 페이지에서 **액세스 키** 메뉴를 선택합니다.

    

2.  삭제할 키가 있는 탭(**API 키**, **LLM API 키**, **S3 키**)을 선택합니다.

3.  삭제할 키 행의 **삭제** 버튼을 클릭합니다.

> **Warning**: 키 삭제 시 주의
>
> 키를 삭제하면 해당 키를 사용하는 모든 외부 연동이 즉시 중단됩니다. 삭제 전에 해당 키를 사용 중인 서비스가 없는지 확인하세요.
