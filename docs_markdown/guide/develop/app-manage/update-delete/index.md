# 애플리케이션 수정 및 삭제

생성된 애플리케이션의 설정을 변경하거나, 더 이상 사용하지 않는 애플리케이션을 삭제하는 방법을 안내합니다.

> 프로젝트 > **애플리케이션** 메뉴

> **Note**: 수정 및 삭제 가능 조건
>
> 애플리케이션이 **배포 해제됨** 상태일 때만 수정하거나 삭제할 수 있습니다.  
> 배포된 애플리케이션은 먼저 [배포 해제](runtime-control.md#stop-app)한 후 수정 또는 삭제할 수 있습니다.

---

## 애플리케이션 설정 변경

배포된 애플리케이션의 설정을 변경합니다. 리소스 할당량, values.yaml 설정 등을 수정하여 애플리케이션 실행 환경을 최적화할 수 있습니다.

1. 프로젝트 화면에서 **애플리케이션** 메뉴로 이동합니다.

    

2. 애플리케이션 목록에서 수정할 애플리케이션 카드를 클릭합니다.

    - 수정하려는 애플리케이션이 **배포됨** 상태인 경우, 배포 해제해야 합니다.

3. 애플리케이션 상세 화면 우측 상단의  (더 보기) > **편집** 메뉴를 클릭합니다.

    
    - 수정하려는 애플리케이션이 **배포됨** 상태인 경우,  (더 보기) 메뉴가 표시되지 않습니다.

4. 애플리케이션 편집 화면에서 필요한 항목을 수정합니다.

        

5. 화면 하단의 **저장** 버튼을 클릭하여 변경 사항을 저장합니다.

---

### 수정 가능한 항목

편집 화면에서 다음 항목들을 수정할 수 있습니다:

**기본 정보:**

- **애플리케이션 이름**: 목록에 표시될 이름
- **설명**: 애플리케이션 용도 및 참고 정보

**구성 (일부 항목만 수정 가능):**

- **헬름 리포지토리 URL**: 변경 불가 (재생성 필요)
- **차트**: 변경 불가 (재생성 필요)
- **차트 버전**: 변경 불가 (재생성 필요)

**헬름 차트:**

- **values.yaml 전체 내용**: 리소스 할당, 스토리지 설정, 환경 변수 등 모든 설정 변경 가능

> **Warning**: 수정 불가 항목
>
> **애플리케이션 ID**는 시스템 내부 고유 식별자로, 생성 후에는 변경할 수 없습니다.
> 헬름 차트 자체(리포지토리, 차트 이름, 버전)를 변경하려면 애플리케이션을 삭제하고 새로 생성해야 합니다.

> **Info**: 볼륨 데이터는 유지됩니다
>
> 애플리케이션 설정을 수정하고 다시 배포해도, **연결된 볼륨의 데이터는 그대로 유지**됩니다.
> 리소스 할당량이나 환경 변수를 변경한 후에도 이전 작업을 이어서 진행할 수 있습니다.

---

## 애플리케이션 삭제

잘못 생성했거나 더 이상 사용하지 않는 애플리케이션을 완전히 제거합니다.

> **Danger**: 삭제 전 반드시 확인하세요
>
> 애플리케이션을 삭제하면 다음 데이터가 **영구적으로 삭제**되며 복구할 수 없습니다:
>
> - 애플리케이션 설정 (values.yaml 포함)
> - 배포 이력 및 메타데이터
> - **연결된 볼륨의 모든 데이터** (코드, 모델, 데이터셋 등)
>
> 삭제 전에 중요한 데이터는 반드시 백업하세요.

1. 프로젝트 화면에서 **애플리케이션** 메뉴로 이동합니다.

    

2. 애플리케이션 목록에서 삭제할 애플리케이션 카드를 클릭합니다.

    - 삭제하려는 애플리케이션이 **배포됨** 상태인 경우, 배포 해제해야 합니다.

3. 애플리케이션 상세 화면 우측 상단의  > **삭제** 메뉴를 클릭합니다.

    

    - 수정하려는 애플리케이션이 **배포됨** 상태인 경우,  (더 보기) 메뉴가 표시되지 않습니다.

4. 삭제 확인 팝업에서 **삭제** 버튼을 클릭하여 애플리케이션을 완전히 삭제합니다.

        

> **Tip**: 일시적으로 사용하지 않는 경우
>
> 애플리케이션을 완전히 삭제하지 않고 일시적으로만 중지하려면, [배포 해제](runtime-control.md#stop-app)를 사용하세요.
> 배포 해제 시 컴퓨팅 리소스는 반환되지만 볼륨 데이터는 유지되어, 나중에 다시 배포할 수 있습니다.
