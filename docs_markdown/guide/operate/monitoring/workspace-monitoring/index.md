## 워크스페이스 리소스 사용 현황

워크스페이스 전체의 리소스 할당 현황과 시간에 따른 사용 추이를 확인합니다.

워크스페이스 화면에서 **모니터링** 메뉴 > **워크스페이스** 탭을 선택합니다.

### 리소스 할당 현황

워크스페이스에 할당된 전체 리소스와 현재 사용 중인 리소스를 비교합니다.

각 리소스 카드는 CPU(cores 단위), 메모리(GiB 단위), 스토리지(GiB 단위), GPU(GPUs 단위)에 대해 다음 정보를 표시합니다.

- **할당률(Allocation rate)**: 용량 대비 리소스 할당 비율
    - 계산식: (Allocated / Capacity) × 100
    - 시각화: 프로그레스 바로 표시
        -  **정상 (Green)**: 70% 미만
        -  **주의 (Yellow)**: 70% 이상 90% 미만
        -  **위험 (Red)**: 90% 이상

- **용량(Capacity)**: 워크스페이스에 할당된 리소스의 최대 총량

- **할당됨(Allocated)**: 현재 워크로드에 예약된 리소스

- **할당 가능(Allocatable)**: 아직 할당되지 않은 여유 리소스
    - 계산식: Capacity - Allocated

- **사용됨(Used)**: 현재 실시간으로 사용되고 있는 리소스 양

### 리소스 사용 추이

기간별 리소스 사용량의 변화 추이를 시각적으로 확인할 수 있습니다.

** 조회 기간 선택**

조회 기간은 다음 단위로 선택할 수 있습니다.

- 1시간, 3시간, 6시간, 12시간
- 1일, 3일, 7일

** 리소스별 그래프**

워크스페이스 단위에서 CPU, Memory, Disk, GPU 리소스의 **용량(Capacity)**, **할당됨(Allocated)**, **사용됨(Used)** 항목을 시간 축 기준으로 비교하여 볼 수 있습니다.

- **CPU (cores)**: 사용 중인 CPU 코어 수
- **Memory (GiB)**: 메모리 사용량
- **Disk (GiB)**: 디스크 사용량
- **GPU (GPUs)**: 사용 중인 GPU 개수

각 그래프는 용량(빨간색), 할당됨(파란색), 사용됨(초록색) 수치를 선형 그래프로 표시하여 전체 용량 대비 현재 리소스 활용도를 직관적으로 파악할 수 있습니다.

> **Info**: 용어 해석 가이드
> - **용량(Capacity)**: 워크스페이스에 할당된 최대 리소스
> - **할당됨(Allocated)**: 실제 워크로드 실행을 위해 예약된 리소스
> - **사용됨(Used)**: 현재 시점에 실시간으로 소비 중인 리소스
>
> 일반적으로 `용량 ≥ 할당됨 ≥ 사용됨` 관계를 가집니다.

> **Note**: 그래프 활용 팁
> - **구간 확대**: 그래프 하단의 시계열 바를 드래그하여 특정 구간을 상세 확대할 수 있습니다.
> - **세부 수치 확인**: 각 시점에 마우스를 올리면 정확한 수치를 확인할 수 있습니다.
> - **패턴 분석**: 시간대별 리소스 사용 패턴을 파악하여 최적화에 활용할 수 있습니다.

## 전체 프로젝트 리소스 사용 현황

워크스페이스 내 모든 프로젝트의 리소스 할당 현황과 사용량을 비교하여 확인합니다.

워크스페이스 화면에서 **모니터링** 메뉴 > **프로젝트** 탭을 선택합니다.

### 쿼터 할당 현황

워크스페이스 전체 리소스 중 각 프로젝트에 쿼터로 할당된 리소스 비율을 확인합니다.

- **지정률(Assignment rate)**: 전체 리소스 중 프로젝트에 할당된 비율
    - 계산식: (Assigned / Capacity) × 100

- **용량(Capacity)**: 워크스페이스 내 전체 리소스 총량

- **지정됨(Assigned)**: 프로젝트에 쿼터로 할당된 리소스 합

- **지정 가능(Assignable)**: 아직 프로젝트에 할당되지 않은 여유 리소스
    - 계산식: Capacity - Assigned

### 프로젝트별 리소스 현황

각 프로젝트별 리소스 사용 상태를 테이블 형태로 정량 비교할 수 있습니다. 정렬 및 검색 기능을 활용해 프로젝트의 리소스 상태를 파악하고, 과다할당 또는 비효율적 사용 패턴을 확인하여 조치를 취하는 데 유용합니다.

각 프로젝트에 대해 다음 항목이 표시됩니다.

- **프로젝트(Project)**: 프로젝트 이름 (클릭하여 상세 현황 확인 가능)
- **CPU**: cores 기준의 리소스 상태
- **Memory**: GiB 기준의 메모리 사용 상태
- **Disk**: GiB 기준의 디스크 사용 상태
- **GPU**: GPU 개수 기준의 할당량 및 사용량

**각 리소스 항목의 세부 정보**

CPU, Memory, Disk, GPU에 대해 각각 다음 정보를 제공합니다.

- **할당률(Allocation rate)**: 할당된 리소스 대비 예약된 리소스의 비율
    - 계산식: (Allocated / Assigned) × 100

- **사용률(Utilization)**: 예약된 리소스 대비 실제 사용량 비율
    - 계산식: (Used / Allocated) × 100

- **지정됨(Assigned)**: 프로젝트에 쿼터로 할당된 리소스

- **할당됨(Allocated)**: 워크로드에 예약된 리소스

- **사용됨(Used)**: 실제 사용 중인 리소스

> **Tip**: 프로젝트 상세 확인
> 프로젝트 이름을 클릭하면 해당 프로젝트의 상세 모니터링 화면으로 이동합니다.  
> 자세한 내용은  **[프로젝트 리소스 모니터링](project-monitoring.md)**을 참고하세요.
