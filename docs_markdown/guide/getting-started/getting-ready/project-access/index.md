# 프로젝트 접근 및 참여

AI·ML 워크로드를 시작하려면 프로젝트 접근 권한이 필요합니다. 이 문서에서는 프로젝트 가입을 요청하는 방법을 안내합니다.

> **Note**: 워크스페이스 가입 여부
>
> 프로젝트에 접근하려면 먼저 하나 이상의 **워크스페이스**에 속해 있어야 합니다.
>
> 일반적으로 사용자는 이미 워크스페이스에 소속된 상태로 Runway에 로그인하게 됩니다.  
> **로그인 후 워크스페이스가 표시되지 않는다면**, 아직 어떤 워크스페이스에도 소속되지 않은 상태일 수 있습니다.
>
> 이 경우 **플랫폼 관리자에게 문의**하여 **워크스페이스 접근 권한을 요청**해야 합니다.

---

## 프로젝트 참여 방법

프로젝트에 참여하는 방법은 세 가지입니다.

| 방법 | 설명 |
|------|------|
| **관리자 추가** | 프로젝트 관리자가 직접 사용자를 프로젝트에 추가합니다. |
| **가입 요청** | 사용자가 프로젝트 참여를 요청하고, 관리자가 승인합니다. |
| **프로젝트 생성** | 사용자가 워크스페이스 내에서 새로운 프로젝트를 생성합니다.(**워크스페이스의 관리자/멤버만 가능**) |

> **Info**: 프로젝트 관리자의 사용자 추가
>
> 프로젝트 관리자는 워크스페이스에 소속된 사용자를 프로젝트에 직접 추가할 수 있습니다.  
> 자세한 내용은 **[프로젝트 사용자 추가하기](../../manage/project/project-users.md#add-project-users)**를 참고하세요.

---

### 프로젝트 가입 요청하기

워크스페이스의 **프로젝트** 메뉴에는 모든 프로젝트 목록이 표시되며, 현재 접속한 계정의 참여 **멤버십**(가입 여부)이 표시됩니다.

> 워크스페이스 > **프로젝트** 메뉴

1. 워크스페이스 초기 화면의 **전체 프로젝트** 목록에서 참여하고자 하는 프로젝트의 이름을 클릭합니다.

    

2. **접근 제한** 팝업에서 **가입 요청** 버튼을 클릭하면 참여 신청이 완료됩니다.

    

> **Info**: 가입 요청 후 절차
>
> - 프로젝트 관리자가 요청을 승인하면 프로젝트에 접근할 수 있습니다. 가입이 승인되면 프로젝트 목록에서 해당 프로젝트의 멤버십이 **비멤버**에서 **멤버**로 변경됩니다. 프로젝트 이름을 클릭하여 프로젝트 화면으로 이동할 수 있습니다.
> - ML/AI 개발 프로세스를 수행하려면 **멤버** 이상의 역할이 필요합니다. 뷰어 역할은 조회만 가능합니다.
> - 부여받은 역할은  **[계정 설정에서 확인](../../manage/account/account-profile.md#view-roles)**할 수 있습니다.

### 프로젝트 생성하기

**워크스페이스 관리자**와 **워크스페이스 멤버**는 워크스페이스 내에서 새로운 프로젝트를 생성할 수 있습니다.  
프로젝트를 생성할 때는 팀의 작업 목적과 필요한 리소스 범위를 고려하여 **이름과 설명을 명확하게 작성**하는 것이 좋습니다.

1.  워크스페이스에서 **프로젝트** 메뉴로 이동합니다. 

    

2.  **전체 프로젝트** 영역 오른쪽에 위치한 **+ 생성** 버튼을 클릭합니다.

3.  오른쪽 슬라이드 패널에서 **일반 세부 정보**를 입력합니다.

    

    - **이름**: 프로젝트 이름을 입력합니다.(최대 128자)

    - **ID**: 입력한 프로젝트 이름에 따라 자동 추천되며, 생성 시에는 수정할 수 있습니다. (3-31자, 영문 소문자, 숫자, 하이픈(-)만 사용 가능)

        > **Info**: 프로젝트 ID 생성 규칙
        > 프로젝트 ID는 프로젝트의 고유 식별자로, **생성 후에는 변경할 수 없으므로** 신중하게 설정해야 합니다.
        >
        > ** &nbsp;ID 구성 방식**
        >
        > - 워크스페이스 ID 및 구분자(최대 9자)가 접두사로 자동 추가됩니다.(변경 불가)
        > - 워크스페이스 ID 뒤에 프로젝트 이름 기반으로 자동 추천되나 프로젝트 생성 시에는 수정 가능합니다.
        >
        > {width=60%}
        >
        > ** &nbsp;ID 규칙**
        >
        > 프로젝트 ID는 Kubernetes 네임스페이스명으로 사용되므로 다음 규칙을 준수해야 합니다.
        >
        > - **길이**: 최소 3자, 최대 31자(입력 가능 값 기준)
        > - **허용 문자**: 소문자 영문(a-z), 숫자(0-9), 하이픈(-)
        > - **시작/끝**: 소문자 또는 숫자 (하이픈 불가)
        >
        > ** &nbsp;권장사항**
        >    
        > 짧고 의미가 명확한 ID로 설정하면 관리가 편리합니다.

    - **설명**(선택): 프로젝트에 대한 간단한 설명을 입력합니다.
